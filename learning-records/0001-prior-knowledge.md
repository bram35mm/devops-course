# Prior knowledge: frontend + partial backend development, zero DevOps

用户自述：前端开发为主，有部分后端开发经验。对 DevOps 没有系统认知，命令行基本会用但不熟练。没有 Docker、CI/CD、云服务、Linux 服务器管理的实操经验。

**Evidence:** 用户自述。在痛点调查中未选择任何具体痛点，表明当前工作中不涉及运维任务，属于纯储备性学习。

**Implications:**
- 起点：需要从最基础的概念（什么是 DevOps、它解决什么问题）和最基本的工具（Linux 命令行、Git 进阶）开始
- 优势：有开发经验意味着理解代码生命周期、版本控制、依赖管理——这些和 DevOps 直接衔接
- 注意：不要假定用户知道任何运维概念（进程管理、网络端口、文件权限、systemd 等），每个新概念都需要从零解释
- 课程顺序：概念地图 → Linux 基础 → Git 进阶 → Docker → CI/CD → 云服务 → Infra-as-Code
