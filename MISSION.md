# Mission: DevOps

## Why
从全栈开发（前端 + 部分后端）转型进入 DevOps / SRE / 平台工程团队。需要系统性地补齐运维、基础设施、自动化方面的知识体系，达到能通过 DevOps 岗位面试、胜任日常工作的水平。

## Success looks like
- 能从零搭建一套完整的 CI/CD 流水线（代码提交 → 构建 → 测试 → 部署）
- 能用 Docker 容器化任意项目，用 Docker Compose 编排多服务
- 能操作至少一个云平台（AWS/阿里云），管理基础网络、计算、存储资源
- 能写 Infra-as-Code（Terraform 或同类工具）声明式管理基础设施
- 能在 Linux 服务器上排查常见故障（网络、进程、磁盘、权限）
- 能通过一轮 DevOps 初级/中级岗位的技术面试

## Constraints
- 每周投入 5–8 小时
- 学习风格：理论 + 实操混合，讲一点练一点
- 当前基础：有开发经验，无运维经验；命令行基本会用但不熟练
- 没有公司内部的 DevOps 环境可用 → 所有实验需在个人机器或云 free tier 上完成

## Out of scope
- 高级 SRE 主题（容量规划、混沌工程、SLI/SLO 体系设计）— 入门后再考虑
- 特定认证考试冲刺 — 知识体系优先，认证是副产品
- 大型分布式系统架构设计 — 先聚焦单服务到多服务的运维链路
